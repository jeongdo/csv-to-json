# 9. 댕글링 포인터 (Dangling Pointer)

댕글링 포인터란 **가리키는 대상이 더 이상 유효하지 않은 포인터**를 말한다.  
주로 동적 메모리 관리 시 발생하며, 메모리 누수나 예기치 않은 동작(Undefined Behavior)을 일으킨다.

---

## 발생하는 상황

### 1. 해제된 메모리를 참조할 때

`delete` 또는 `delete[]`로 메모리를 해제한 후, 포인터가 여전히 그 주소를 가리키는 경우.

```cpp
int* p = new int(5);
delete p;
// p는 이제 댕글링 포인터
*p = 10;  // Undefined Behavior
```

---

### 2. 스택 메모리 참조

함수가 종료된 후에도 지역 변수를 가리키는 포인터가 남아있는 경우.  
함수가 끝나면 지역 변수는 스택에서 소멸되는데, 그 주소를 반환하면 댕글링 포인터가 된다.

```cpp
int* danglingPointer() {
    int localVar = 10;
    return &localVar;  // localVar은 함수 종료 시 소멸
}

int main() {
    int* p = danglingPointer();
    // p는 이제 댕글링 포인터
    *p = 5;  // Undefined Behavior
    return 0;
}
```

> 버퍼 오버플로우와 연결되는 개념 — 스택에서 소멸된 메모리를 건드리는 것 자체가 위험

---

### 3. 객체가 소멸된 후 참조

객체를 `delete`한 후, 그 객체의 멤버 포인터가 남아있는 경우.

```cpp
class MyClass {
public:
    int value;
};

int main() {
    MyClass* obj = new MyClass();
    int* p = &obj->value;
    delete obj;
    // obj와 멤버 value 모두 무효화됨
    *p = 10;  // Undefined Behavior
    return 0;
}
```

---

## 방지하는 방법

### 1. 해제 후 nullptr 설정

메모리 해제 즉시 포인터를 `nullptr`로 초기화.

```cpp
int* p = new int(5);
delete p;
p = nullptr;  // 이후 실수로 접근해도 null 체크로 방어 가능
```

---

### 2. 스마트 포인터 사용 (C++11 이후 권장)

`std::unique_ptr`, `std::shared_ptr`를 사용하면 메모리가 자동 관리된다.

```cpp
#include <memory>

int main() {
    std::unique_ptr<int> p = std::make_unique<int>(5);
    // 스코프 벗어나면 자동 해제 → 댕글링 포인터 원천 차단
    return 0;
}
```

---

### 3. RAII 원칙 적용

객체의 생성자에서 자원 할당, 소멸자에서 자원 해제.  
자원의 수명을 객체의 수명과 일치시키는 패턴.

```cpp
#include <iostream>

class Resource {
public:
    Resource()  { std::cout << "Resource acquired\n"; }
    ~Resource() { std::cout << "Resource released\n"; }
};

void useResource() {
    Resource res;
    // 함수 종료 시 res 소멸자 자동 호출 → 자원 자동 해제
}

int main() {
    useResource();
    return 0;
}
```

---

## 요약

| 상황 | 문제 | 해결 |
|------|------|------|
| `delete` 후 포인터 사용 | 해제된 메모리 접근 | 해제 후 `nullptr` 설정 |
| 지역변수 주소 반환 | 스택 소멸 후 접근 | 반환하지 않거나 동적 할당 사용 |
| 객체 삭제 후 멤버 접근 | 무효화된 메모리 접근 | 스마트 포인터 / 수명 관리 |

> 댕글링 포인터는 버퍼 오버플로우와 함께 C/C++의 대표적인 메모리 취약점.  
> 현대 C++에서는 raw 포인터 대신 스마트 포인터를 쓰는 것이 기본 원칙.
