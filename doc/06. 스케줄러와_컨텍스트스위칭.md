# 스케줄러와 컨텍스트 스위칭

## 시작점

프로세스와 스레드를 이해했다.

그런데 의문이 생긴다.

> CPU가 1개인데 어떻게 여러 프로그램이 동시에 실행되는 것처럼 보일까?

답은 스케줄러와 컨텍스트 스위칭이다.

---

# 1. CPU는 한 번에 하나만 실행한다

CPU 코어 하나는 특정 순간에

```text
프로세스 A
```

또는

```text
프로세스 B
```

만 실행할 수 있다.

동시에 둘 다 실행할 수는 없다.

---

# 2. 스케줄러의 역할

커널 안에는 스케줄러가 존재한다.

```text
프로세스 A
프로세스 B
프로세스 C
```

중 누가 CPU를 사용할지 결정한다.

```text
A 10ms
 ↓
B 10ms
 ↓
C 10ms
 ↓
A 10ms
```

매우 빠르게 전환한다.

---

# 3. 왜 동시에 실행되는 것처럼 보일까

사람은 수 ms 단위 변화를 인식하지 못한다.

그래서

```text
A
B
C
```

가 매우 빠르게 교대 실행되면

동시에 실행되는 것처럼 느낀다.

---

# 4. 컨텍스트란 무엇인가

프로세스가 실행되던 상태를 의미한다.

예)

```text
레지스터 값
프로그램 카운터
스택 정보
```

등.

---

# 5. 컨텍스트 스위칭

프로세스 A 실행 중

```text
A 실행
 ↓
상태 저장
 ↓
B 상태 복원
 ↓
B 실행
```

이 과정을 컨텍스트 스위칭이라고 한다.

---

# 6. MCU 인터럽트와의 유사성

MCU에서

```text
현재 작업
 ↓
인터럽트 발생
 ↓
ISR 실행
 ↓
원래 작업 복귀
```

를 경험했다.

컨텍스트 스위칭도 비슷하다.

```text
현재 작업 저장
 ↓
다른 작업 실행
 ↓
원래 작업 복귀
```

---

# 7. 타이머 인터럽트

스케줄러는 보통 타이머 인터럽트를 이용한다.

```text
타이머 인터럽트
 ↓
커널 진입
 ↓
스케줄러 실행
 ↓
프로세스 전환 여부 결정
```

즉,

CPU 시간 관리도 결국 인터럽트 기반이다.

---

# 최종 그림

```text
타이머 인터럽트
 ↓
커널
 ↓
스케줄러
 ↓
컨텍스트 스위칭
 ↓
프로세스 실행
```

인터럽트가 I/O를 처리했다면,

스케줄러는 CPU 시간을 관리한다.

둘 다 결국 커널이 수행하는 핵심 기능이다.
