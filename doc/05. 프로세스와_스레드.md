# 프로세스와 스레드

## 시작점

이전 글에서 시스템콜을 이해했다.

```text
프로그램
 ↓
시스템콜
 ↓
커널
```

그런데 새로운 의문이 생긴다.

> 프로그램은 어떻게 실행되는가?

> 크롬, IntelliJ, 오라클은 어떻게 동시에 동작하는가?

이 질문의 답이 프로세스와 스레드다.

---

# 1. 프로그램과 프로세스

프로그램은 디스크에 저장된 파일이다.

```text
chrome.exe
java.exe
go.exe
```

아직 실행되지 않았다.

실행되는 순간

```text
프로그램
 ↓
메모리 적재
 ↓
프로세스 생성
```

이 된다.

즉,

> 프로세스 = 실행 중인 프로그램

이다.

---

# 2. MCU와의 차이

MCU 시절에는 보통

```c
int main()
{
    while(1)
    {
    }
}
```

하나가 전부였다.

```text
프로그램 1개
CPU 1개
```

하지만 PC는

```text
크롬
카카오톡
오라클
IntelliJ
```

가 동시에 실행된다.

각각이 프로세스다.

---

# 3. 프로세스는 독립된 공간을 가진다

프로세스마다

```text
메모리
파일 핸들
환경 변수
```

등을 가진다.

```text
프로세스 A
 └─ 독립 공간

프로세스 B
 └─ 독립 공간
```

그래서 한 프로세스가 죽어도 다른 프로세스는 계속 동작할 수 있다.

---

# 4. 스레드의 등장

프로세스 안에서도 여러 작업을 동시에 하고 싶다.

예)

```text
크롬
 ├─ 화면 렌더링
 ├─ 네트워크 통신
 └─ 사용자 입력 처리
```

여기서 각각의 작업자가 스레드다.

---

# 5. 스레드는 같은 집에 사는 가족

프로세스는 집이다.

스레드는 그 집에 사는 가족이다.

```text
프로세스
 ├─ 스레드 1
 ├─ 스레드 2
 └─ 스레드 3
```

스레드들은 메모리를 공유한다.

그래서 빠르지만 동기화 문제가 생길 수 있다.

---

# 6. Java와 스레드

```java
new Thread(...).start();
```

를 호출하면

결국 커널에게 스레드 생성을 요청한다.

```text
Java
 ↓
JVM
 ↓
시스템콜
 ↓
커널
 ↓
스레드 생성
```

---

# 최종 그림

```text
프로그램
 ↓
프로세스
 ↓
스레드
```

프로세스는 실행 단위,

스레드는 작업 단위다.
