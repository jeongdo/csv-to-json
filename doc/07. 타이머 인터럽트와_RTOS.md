# 7. 타이머 인터럽트와 RTOS

## 1. 타이머 인터럽트란

타이머 인터럽트는 CPU가 실행 중일 때 **하드웨어 타이머가 주기적으로
신호를 보내 CPU 실행 흐름을 잠시 끊고 커널로 제어를 넘기는 이벤트**이다.

핵심은 다음과 같다:

-   CPU가 특정 작업을 하고 있어도 발생
-   특정 프로세스에 종속되지 않음
-   시스템 전체 기준으로 동작

------------------------------------------------------------------------

## 2. 타이머 인터럽트의 주체는 누구인가

주체는 "프로세스"가 아니라 **하드웨어 타이머 장치**이다.

대표적인 구성:

-   PIT (Programmable Interval Timer)
-   APIC Timer (현대 CPU)
-   HPET (High Precision Event Timer)

이 장치들은 메인보드/CPU 내부 클럭을 기반으로 동작한다.

------------------------------------------------------------------------

## 3. 왜 주기적으로 발생하는가

OS가 부팅 시 다음을 설정한다:

-   인터럽트 주기 (예: 1ms, 10ms 등)
-   타이머 카운트 값

이 설정 이후에는:

> 하드웨어 타이머가 독립적으로 계속 카운트 → 시간이 되면 인터럽트 발생

즉, 특정 작업이 계속 실행 중이라서 발생하는 것이 아니라\
**시간 기반 독립 장치가 계속 발생시키는 구조**이다.

------------------------------------------------------------------------

## 4. 타이머 인터럽트 발생 조건

조건은 단순하다:

-   설정된 시간 간격 도달
-   CPU 상태와 무관

즉:

-   CPU가 바쁘든
-   idle 상태든
-   어떤 프로세스가 실행 중이든

무조건 발생한다.

------------------------------------------------------------------------

## 5. RTOS와 일반 OS의 차이

### 일반 OS

-   타이머 인터럽트 = 스케줄링 기회
-   실행 순서 유동적
-   지연 허용

### RTOS

-   타이머 + 스케줄링이 예측 가능해야 함
-   실행 시간 보장 (deterministic)
-   인터럽트 지연 최소화

------------------------------------------------------------------------

## 6. 핵심 구조 요약

타이머 인터럽트 흐름:
```text
CPU 실행 중 
    ↓ 
하드웨어 타이머 카운트 완료 
    ↓ 
인터럽트 발생 
    ↓ 
커널 진입 
    ↓
스케줄러 판단 
    ↓ 
필요 시 컨텍스트 스위치 
    ↓ 
아니면 기존 프로세스 복귀
```

------------------------------------------------------------------------

## 7. 핵심 개념 정리

-   타이머 인터럽트는 "프로세스 기반"이 아니라 "시간 기반"
-   OS는 이 이벤트를 이용해 CPU 점유를 관리
-   RTOS는 이 구조 위에서 "시간 보장성"을 강화한 시스템
